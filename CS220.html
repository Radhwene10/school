Introduction to Web Development

What is the Web

The basic idea of the WWW was to merge the evolving technologies of computers, data networks and hypertext into a powerful and easy to use global information system

Web VS Internet

Aspect Internet World Wide Web (Web)
Definition The internet is a vast global network that connects millions of interconnected devices worldwide, including computers, servers, routers, and more. The Web, short for the World Wide Web, is a subset of the internet. It's a system of interconnected documents and resources linked by hyperlinks and accessible via web browsers.
Infrastructure Physical and virtual infrastructure, including cables, satellites, data centers, and network protocols, that facilitate data transmission and communication across devices. A system of information presentation and retrieval that utilizes the internet's infrastructure to provide access to documents and resources.
Examples Examples of internet services include email, FTP (File Transfer Protocol), VoIP (Voice over Internet Protocol), online gaming, and more. Examples of web services include websites, web applications, online stores, search engines, social media platforms, and blogs.

What is Web Development

The process of creating, building, and maintaining websites and web applications for the internet.

It encompasses a wide range of tasks and skills aimed at producing interactive, functional, and visually appealing online experiences.

Web Key Components

  1. URL: Uniform Resource Locator

    2. Specify the exact location of a resource on the internet. It includes information on how to access that resource Specific to web resources and are used to identify and locate online content. They are commonly used in web browsers to access websites and web pages

    Components:
    • Protocol: Specifies the method or protocol used to access the resource (e.g., "http://" or "https://").
    • Domain Name or IP Address: Identifies the location of the server hosting the resource (e.g., "www.example.com" or an IP address).
    • Port (Optional): Specifies the port number on the server (e.g., ":80" for HTTP).
    • Path: Indicates the specific location of the resource on the server (e.g., "/page.html").
    • Query (Optional): Provides additional parameters for accessing the resource (e.g., "?id=123").
    • Fragment (Optional): Points to a specific section or anchor within the resource (e.g., "#section2").
  2. HTTP: HyperText Transfer Protocol
    3. Class Code Text Comments
    Success 200 OK All is well, valid request and response.
    Redirection 301 Moved Permanently Page is now at a new location, automatic redirects built in to most browsers.
    302 Found Page is now at a new location temporarily.
    Client Error 401 Unauthorized Page typically requires login credentials.
    403 Forbidden Server will not allow this request.
    404 Not Found Server cannot find what was asked for.
    Server Error 500 Internal Server Error Generic server failure in responding to the otherwise-valid request.

    Dictates the format by which clients request web pages from a server, and the format via which servers return information to clients

  3. HTML: HyperText Markup Language

    4. A markup language Use angle-bracket enclosed tags to semantically define the structure of a web page
    Causing the plain text inside of sets of tags to be interpreted by web browsers in different ways.
